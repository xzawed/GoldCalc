# 의사결정 필요 사항 (은시세 + 국내금시세 확장)

아래 항목은 개발자가 직접 확인·처리해야 하는 사항입니다.
각 항목 완료 시 체크(`[x]`)하고, 필요한 값을 `.env`에 반영해 주세요.

---

## 1. 은시세 API (gold-api.com)

- [ ] **gold-api.com 엔드포인트 동작 확인**
  - 브라우저에서 `https://api.gold-api.com/price/XAG` 접속하여 JSON 응답 확인
  - 히스토리: `https://api.gold-api.com/history?metal=XAG` 응답 스키마 확인
  - CORS 정상 동작 여부 확인 (브라우저 콘솔에서 fetch 테스트)

- [ ] **응답 스키마 확인 후 코드 반영**
  - 현재 코드는 예상 스키마 기반으로 작성됨 → 실제 응답과 다르면 훅 수정 필요
  - 특히 히스토리 엔드포인트의 날짜 파라미터 형식, 응답 필드명 확인

- [ ] **GoldAPI.io 잔여 호출 한도 확인**
  - 현재 금시세에 사용 중인 GoldAPI.io 무료 플랜의 월별 호출 제한 확인
  - 은시세를 gold-api.com으로 분리했으므로 기존 한도에는 영향 없음

> **환경변수**: gold-api.com은 인증 불필요 → `.env` 추가 불필요

---

## 2. 국내 금시세 API (data.go.kr)

- [ ] **공공데이터포털 회원가입**
  - https://www.data.go.kr 접속 → 회원가입

- [ ] **금시세 API 활용 신청**
  - API명: `금융위원회_일반상품시세정보` (GetGeneralProductInfoService)
  - URL: https://www.data.go.kr/data/15094805/openapi.do
  - "활용신청" 버튼 클릭 → 사용 목적 입력 → 키 발급 (보통 즉시~수 시간)

- [ ] **발급받은 serviceKey를 `.env`에 추가**
  ```
  VITE_DATA_GO_KR_API_KEY=발급받은_서비스키
  ```

- [ ] **API Swagger 확인 후 응답 스키마 공유**
  - 활용신청 후 Swagger UI에서 금시세 관련 오퍼레이션 확인
  - 요청 파라미터(날짜 형식, 페이지네이션 등)와 응답 필드 확인
  - 확인된 스키마를 공유해 주시면 프록시 및 훅 코드 완성

- [x] **Railway 환경변수 등록**
  - Railway 대시보드 → 서비스 → Variables
  - `DATA_GO_KR_API_KEY` 추가 (변수 저장 즉시 재배포 자동 적용)

## 3. UI/브랜딩 결정

- [ ] **앱 이름 변경 여부**
  - 현재: "GoldCalc 국제 금시세 계산기"
  - 제안: "GoldCalc 귀금속 시세 계산기" 또는 그대로 유지
  - 결정: ____________________

- [ ] **은 순도 지원 범위**
  - 현재 코드에 포함: 999 (순은 99.9%), 925 (스털링 실버), 900, 800
  - 불필요한 순도가 있으면 제거, 추가할 순도가 있으면 공유

## 4. 국내금시세 세부 기능 결정

- [ ] **매매 구분 UI 필요 여부**
  - 살 때(매도가) / 팔 때(매입가) 구분을 표시할지
  - data.go.kr API가 매매 구분 데이터를 제공하는지 Swagger에서 확인 필요

- [ ] **세금(부가세 10%) 포함 표시 여부**
  - KRX 금시장 가격에 부가세 포함/미포함 토글을 넣을지

- [ ] **국내금시세 예측 기능 포함 여부**
  - 국제금과 동일하게 예측 차트를 제공할지
  - 국내금은 환율 변동 + 국제 금시세 이중 변수라 예측 정확도가 낮을 수 있음
